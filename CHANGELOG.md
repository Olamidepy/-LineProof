# Changelog

All notable changes to this project are documented in this file.

This project follows the spirit of Keep a Changelog and uses semantic versioning once tagged releases begin.

## Unreleased

### Added

- Soroban contract workspace for queue, queue factory, enrollment, identity, and escrow components.
- TypeScript SDK package structure with queue, enrollment, identity, escrow, and shared client modules.
- Reference frontend and backend application directories.
- Example application directories for healthcare, visa appointments, event ticketing, and product launches.
- Documentation for architecture, system overview, queue lifecycle, anti-scalping, escrow, security, testing, deployment, SDK architecture, and developer onboarding.
- Research notes for healthcare waitlists, visa appointment systems, university admissions, event ticketing, and limited product releases.
- Project planning documents: roadmap, governance, changelog, and contribution guidelines.

### Changed

- Replaced unclear `fe` and `be` directory names with `frontend` and `backend`.
- Clarified that current advanced fairness, production identity verification, decentralized governance, and formal verification work are planned rather than complete.
- Expanded maintainer documentation around trust boundaries and production-readiness gaps.

### Security

- Documented threat model, residual risks, and review triggers.
- Documented escrow release, refund, and expiry invariants.
- Documented privacy guidance for avoiding personal data on-chain.
